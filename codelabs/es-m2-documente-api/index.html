Como se demostró , publicar su API en Exchange es tan fácil como unos pocos clics. Sin embargo, Exchange acaba de capturar los atributos básicos y la documentación para su API en este momento. Asegurarse de que su API sea fácil de encontrar y "lista para el autoservicio" es su responsabilidad como diseñador de API. Busquemos nuestra API en Exchange y agreguemos más a nuestra documentación de API.

  1. Navegue a Exchange desde el "menú de hamburguesa" en la esquina superior izquierda de Design Center.

  1. Utilice la barra de búsqueda para encontrar su AP,I si no está fácilmente visible en la lista de activos de Exchange. Haga clic en su API una vez que la encuentre:
  2. Haga clic en cualquiera de los botones "Edit" para iniciar el modo de edición.
    Cree su propia documentación para su API o copie el siguiente texto en el editor:
** Omni Channel Experience API * * se utiliza para simplificar el acceso a los pedidos, los productos y las funciones del carrito de la compra necesarios para las aplicaciones móviles y de comercio electrónico.

Los desarrolladores que lo requieran pueden componer fácilmente aplicaciones innovadoras aprovechando esta API para hacer lo siguiente:

 - Recuperar el catálogo de productos y los detalles sobre un producto
 - Agregar productos a un carrito de compras con estado
 - Buscar pedidos y recuperar su estado
 - Crear un nuevo pedido



  1. Haga clic en el botón "Save as draft" en la parte inferior de la pantalla. Su borrador de la documentación de la API se guarda pero no se publica hasta que "Publica" formalmente las actualizaciones finales en Exchange, así que siéntase libre de guardar varios borradores mientras crea su documentación.
  • Puede presionar Publish, Discard draft o Exit draft.
  • Publish: publicar cambios en la documentación para intercambiarlos.
  • Discard draf: desechar los cambios.
  • Exit draft: mantener los cambios, pero no publicarlos en Exchange.
  1. Presione el botón Publish. ¡Sus cambios en la documentación ahora son visibles en Exchange!

Al igual que en el Design Center, en Exchange puede consultar el API Summary. El API Summary le brinda la oportunidad de realizar una solicitud y llamar al servicio simulado o una implementación de la API.

Vamos a ver cómo se puede hacer.

  1. Vaya al API Summary
  2. Presione /products/search [GET]
  3. En el lado derecho del panel verá el método GET al que se llamará.
    Puede seleccionar de una lista combinada de puntos finales de implementación disponibles para llamar. En este caso solo tenemos el servicio simulado y lo que se define en la especificación.Puede ver los parámetros y encabezados cómo se completan.
    En la pestaña parameters, opcionalmente, puede hacer clic en la casilla de verificación y ver todos los parámetros opcionales que se pueden enviar.
  4. Presione Send y vea qué sucede.
